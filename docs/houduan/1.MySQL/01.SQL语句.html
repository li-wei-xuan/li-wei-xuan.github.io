# SQL语句

SQL (结构化查询语言)是用于执行查询的语法。
SQL语言也包含用于更新、插入和删除记录的语法。
注意：一定要记住，SQL 对大小写不敏感

# 语法

# SELECT(查询语句)

查询全部字段
    SELECT * FROM `表`

查某个字段
    SELECT 字段1,字段2 FROM 表

条件查询
    SELECT * FROM `表` WHERE 条件

AND
    SELECT * FROM `表` WHERE 条件 and 条件

OR
    SELECT * FROM `表` WHERE 条件 or 条件

BETWEEN
    SELECT * FROM `表` WHERE `id` BETWEEN 5 AND 10
    SELECT * FROM `表` WHERE `id` >= 5 AND `id` <= 10

IN  
    SELECT * FROM `表` WHERE `id` IN (1,3,6)

NOT IN
    SELECT * FROM `表` WHERE `id` NOT IN (1,3,6)

LIMIT(start,end)
    SELECT * FROM `表` LIMIT 5

从第4条数据开始，只查询5条
    SELECT * FROM `表` LIMIT 3,5

排序（升序：asc，降序：desc）
    SELECT * FROM `表` ORDER BY `id` DESC
    SELECT * FROM `表` ORDER BY `id` ASC

GROUP BY
    SELECT * FROM `表` GROUP BY title

like
    SELECT * FROM `表` WHERE `name` like '%XXX%'

    INSERT INTO `表名`(`字段1`,`字段2`)VALUES('值1'，‘值2’),('值1','值2')

# 连表查询

# LEFT JOIN(左连表)

以左边的表为基准，先把左边的表查出来，再查右边的表，若右边的表没对应的数据显示的就为NULL。

例：
    SELECT a.id,b.name FROM `user` as a LEFT JOIN `job` as b on a.id = b.user_id

以右边的表为基准，先把右边的表查出来，再查左边的表，若左边的表没对应的数据显示的就为NULL。

# INNER JOIN(内连表)

以两个表中有至少一个匹配，否则不显示
这说明inner join并不以谁为基础,它只显示符合条件的记录。

例：
    SELECT a.id,b.name FROM `user` as a INNER JOIN `job` as b on a.id = b.user_id
